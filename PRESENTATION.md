# DriverTRACK
## نظام تتبع حضور السائقين بالذكاء الاصطناعي
### Driver Attendance Tracking with QR Code

---

## 1. المشكلة / Problem

- **صعوبة تتبع حضور وانصراف السائقين** في شركات التوصيل والنقل
- الاعتماد على **السجلات الورقية** أو جداول Excel غير الموثوقة
- **التزوير** في تواقيت الحضور (تسجيل حضور نيابة عن السائق)
- **عدم وجود تقارير فورية** للإدارة عن الحضور والانصراف
- صعوبة التحقق من هوية السائق عند التسجيل

---

## 2. الحل / Solution

- **تطبيق ويب متكامل** يعمل على جميع الأجهزة (جوال، كمبيوتر، تابلت)
- كل سائق يملك **رمز QR خاص به** يتغير يومياً ومشفر تشفيراً قوياً
- **ماسح ضوئي (دوشيت / Douchette)** يقرأ الرمز ويسجل الحضور بضغطة زر
- **تسجيل حضور فوري** مع منع التكرار (لا يمكن تسجيل الحضور مرتين في نفس اليوم)
- **توزيع الصلاحيات**:
  - `Admin` → إدارة كاملة (سائقين، مشرفين، تقارير)
  - `OPS` → مسح QR وإضافة سائقين جدد
  - `Driver` → عرض رمزه QR وتاريخ حضوره فقط

## 3. المعمارية / Architecture

```
┌─────────────────────────────────────────────────┐
│                   Frontend                       │
│          React + Vite (PWA)                      │
│    Mobile-first UI / RTL (Arabic)                │
└──────────────┬──────────────────────┘
               │ REST API (JSON)
               ▼
┌─────────────────────────────────────────────────┐
│                   Backend                        │
│          Node.js + Express                       │
│    JWT Auth | HMAC-SHA256 QR | Role-based ACL   │
└──────────────┬──────────────────────┘
               │ SQL Queries
               ▼
┌─────────────────────────────────────────────────┐
│              PostgreSQL Database                 │
│          Render Free Tier (1GB)                  │
│    Tables: users, attendance                     │
└─────────────────────────────────────────────────┘
```

### تدفق العمل / Workflow

```
1. Admin يضيف سائق جديد → 2. السائق يسجل دخول
   → 3. يفتح رمز QR الخاص به ← 4. مشرف OPS يمسح الرمز
   → 5. تسجيل الحضور في قاعدة البيانات
```

## 4. الميزات / Features

### 👑 Admin
| الميزة | الوصف |
|--------|-------|
| لوحة التحكم الرئيسية | إحصائيات فورية (عدد السائقين، الحضور اليومي) |
| إدارة السائقين | إضافة / تعديل / حذف / تفعيل وتعطيل |
| إدارة مشرفي OPS | إنشاء حسابات مشرفين جدد |
| سجل الحضور | تصفية حسب التاريخ والسائق، تقارير كاملة |
| لوحة الحضور اليومي | عرض من حضر ومن لم يحضر |

### 🛠️ OPS (Operations)
| الميزة | الوصف |
|--------|-------|
| مسح QR | إدخال تلقائي عبر الدوشيت أو يدوي |
| إضافة سائقين | إنشاء حسابات سائقين جدد |
| عرض السائقين | قائمة بجميع السائقين |
| سجل الحضور | عرض تقارير الحضور (قراءة فقط) |

### 🚗 Driver (سائق)
| الميزة | الوصف |
|--------|-------|
| رمز QR اليومي | يظهر مرة واحدة فقط ولا يمكن إعادة عرضه |
| سجل الحضور | آخر 50 تسجيل حضور خاص به |
| تسجيل خروج | العودة لشاشة تسجيل الدخول |

## 5. الأمان / Security

- **JWT Tokens**: صلاحية 24 ساعة لكل جلسة
- **HMAC-SHA256**: توقيع QR code مشفر بمفتاح سري (QR_SECRET)
- **رمز QR صالح ليوم واحد فقط** — ينتهي في منتصف الليل
- **منع التلاعب**: التوقيع الرقمي يمنع تزوير QR أو إعادة استخدامه
- **صلاحيات صارمة**: كل دور لا يمكنه تجاوز صلاحياته
- **كلمات مرور مشفرة**: bcryptjs (10 rounds salt)
- **HTTPS**: عبر Render (شهادة SSL تلقائية)

## 7. دليل الاستخدام / User Guide

### تسجيل الدخول
```
URL: https://drivertrack.onrender.com

Admin:   admin    / Admin@123
OPS:     ops1     / Ops@123
Driver:  driver1  / Driver@123
```

### إضافة سائق جديد (Admin / OPS)
1. سجل دخول كـ Admin أو OPS
2. اذهب إلى **Drivers Management**
3. اضغط **Add Driver**
4. أدخل البيانات: الاسم، اسم المستخدم، كلمة المرور
5. اضغط **Create**

### مسح حضور (OPS)
1. سجل دخول كـ OPS
2. اذهب إلى **Scan QR**
3. وجه الدوشيت نحو QR code السائق
4. تسمع صوت بيب → تم التسجيل ✅

### عرض تقارير الحضور (Admin / OPS)
1. اذهب إلى **Attendance Logs**
2. صفي حسب التاريخ أو السائق
3. اعرض جميع التسجيلات
